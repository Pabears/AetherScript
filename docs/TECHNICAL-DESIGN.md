# AetherScript Technical Design

> Version: 2.0 | Status: Finalized | Date: 2026-03-04

## Overview

AetherScript is a TypeScript code generation framework driven by `@autogen`-annotated abstract classes. The pipeline scans source files, sanitizes prompts, calls an LLM, validates output, and writes results atomically. Only `AbstractCodeGenerator` is itself `@autogen`; all other components are hand-implemented for safety and reliability.

---

## Data Flow

```
ScanResult
  → AbstractPromptSanitizer → SanitizedPayload (runtime Symbol-verified)
    → AbstractLLMClient → UntrustedGenerationResult
      → AbstractValidator → ValidatedResult
        → AbstractFileWriter → written to disk
```

All stage types are **branded** — you cannot pass raw strings between stages.

## Branded Types

```typescript
// Scan output
interface ScanResult {
  filePath: string;
  className: string;
  methodSignatures: MethodSignature[];
  typeContext: string;
}

interface MethodSignature {
  name: string;
  parameters: ParameterDef[];
  returnType: string;
  jsDoc?: string;
}

interface ParameterDef {
  name: string;
  type: string;
  optional?: boolean;
}

// Brand symbol — runtime verification, not just compile-time
declare const SanitizedBrand: unique symbol;
interface SanitizedPayload {
  readonly [SanitizedBrand]: true;
  prompt: string;
  scanResult: ScanResult;
  sanitizedAt: number; // unix ms
}

// Raw LLM output — untrusted until validated
interface UntrustedGenerationResult {
  rawCode: string;
  model: string;
  tokensUsed: TokenUsage;
  generatedAt: number;
}

interface TokenUsage {
  input: number;
  output: number;
  totalCost: number; // USD
}

// Post-validation — safe to write
interface ValidatedResult {
  code: string;
  contractVerified: boolean;
  dangerousAPIsFound: string[]; // empty = clean
  compilationPassed: boolean;
  scanResult: ScanResult;
}
```

## Component Interfaces

### 1. AbstractScanner

Scans TypeScript source files for `@autogen`-annotated abstract classes.

**Cannot be autogen.**

```typescript
interface IScanner {
  scan(projectDir: string): Promise<ScanResult[]>;
  parseFile(filePath: string): Promise<ScanResult[]>;
}
```

---

### 2. AbstractPromptSanitizer

Sanitizes user-supplied and scanned content before passing to LLM. Defends against prompt injection.

**Cannot be autogen. Absolute security boundary — never let an LLM generate its own sanitizer.**

```typescript
interface IPromptSanitizer {
  sanitize(raw: ScanResult): Promise<SanitizedPayload>;
  // Returns a SanitizedPayload with runtime Symbol brand set
}
```

---

### 3. AbstractLLMClient

**Runtime environment abstraction.** The interface decouples the pipeline from any specific LLM provider or runtime. Different environments implement this differently — no API keys, HTTP clients, or provider SDKs leak into the interface.

**Cannot be autogen.** Also the primary mock boundary for testing.

```typescript
interface ILLMClient {
  generate(payload: SanitizedPayload): Promise<UntrustedGenerationResult>;
  getModel(): string;
}
```

**Implementation examples by environment:**

| Environment | Implementation | Notes |
|---|---|---|
| OpenClaw | `openclaw-llm-client.impl.ts` | Uses OpenClaw's built-in model runtime directly; no API key, no HTTP |
| Standalone Node.js | `anthropic-llm-client.impl.ts` | Uses Anthropic SDK + `ANTHROPIC_API_KEY` |
| OpenAI-compatible | `openai-llm-client.impl.ts` | Uses OpenAI SDK or any compatible endpoint |
| Testing | `mock-llm-client.impl.ts` | Returns fixture responses; no network calls |

**Interface constraints** (must not leak implementation details):
- No `apiKey`, `baseUrl`, `model` parameters in `generate()` — those belong in the constructor/config
- No HTTP-specific error types in the return signature
- `getModel()` returns a human-readable string for logging only; callers must not branch on it

### 4. AbstractCodeGenerator ✅ @autogen

The **only** class that is itself generated by the framework.

```typescript
interface ICodeGenerator {

  buildPrompt(payload: SanitizedPayload): string;
}
```

### 5. AbstractTokenTracker

Tracks token usage and enforces cost circuit breakers.

**Cannot be autogen.**

```typescript
interface ITokenTracker {
  record(usage: TokenUsage): void;
  getTotalCost(): number;
  getSessionUsage(): TokenUsage;
  checkBudget(): BudgetStatus;
  reset(): void;
}

type BudgetStatus =
  | { ok: true }
  | { ok: false; reason: 'session_limit' | 'run_limit'; cost: number; limit: number };
```

---

### 6. AbstractValidator

Three-in-one validation gate. All three checks must pass for output to proceed.

**Cannot be autogen.**

```typescript
interface IValidator {
  // Check: generated code implements all methods from ScanResult
  verifyContract(result: UntrustedGenerationResult, scanResult: ScanResult): Promise<ContractCheckResult>;

  // Check: no dangerous APIs (fs.unlink, exec, eval, etc.) — patterns externally configurable
  scanDangerousAPIs(result: UntrustedGenerationResult): Promise<DangerousAPICheckResult>;

  // Check: TypeScript compilation passes
  checkCompilation(result: UntrustedGenerationResult): Promise<CompilationCheckResult>;

  // Runs all three; returns ValidatedResult or throws SecurityGateError
  validate(result: UntrustedGenerationResult, scanResult: ScanResult): Promise<ValidatedResult>;
}

interface ContractCheckResult {
  passed: boolean;
  missingMethods: string[];
}

interface DangerousAPICheckResult {
  passed: boolean;
  found: string[];
}

interface CompilationCheckResult {
  passed: boolean;
  errors: string[];
}

// Thrown by validate() when any check fails — blocks FileWriter
class SecurityGateError extends Error {
  constructor(
    public readonly stage: 'contract' | 'dangerous_apis' | 'compilation',
    public readonly detail: ContractCheckResult | DangerousAPICheckResult | CompilationCheckResult
  ) { super(...); }
}
```

**Dangerous API patterns are externally configurable:**

```typescript
interface ValidatorConfig {
  dangerousAPIPatterns: RegExp[];
  // defaults: [/eval\(/, /exec\(/, /fs\.unlink/, /process\.exit/, ...]
}
```

---

### 7. AbstractFileWriter

Atomic transactional write + DI container AST merge.

**Cannot be autogen.**

```typescript
interface IFileWriter {
  // Writes implementation file atomically (tmp → rename)
  write(result: ValidatedResult): Promise<WriteResult>;

  // Merges new class binding into container.ts via AST
  updateContainer(result: ValidatedResult): Promise<void>;
}

interface WriteResult {
  filePath: string;
  bytesWritten: number;
  containerUpdated: boolean;
}
```

---

### 8. AbstractPipeline

Orchestrates the full flow. Calls components in order, enforces SecurityGate, reports progress.

**Cannot be autogen.**

```typescript
interface IPipeline {
  run(projectDir: string): Promise<PipelineResult>;
  runSingle(scanResult: ScanResult): Promise<SingleRunResult>;
}

interface PipelineResult {
  total: number;
  succeeded: number;
  failed: number;
  results: SingleRunResult[];
  totalCost: number;
}

interface SingleRunResult {
  scanResult: ScanResult;
  status: 'success' | 'security_gate' | 'validation_failed' | 'llm_error' | 'write_error';
  output?: ValidatedResult;
  error?: Error;
  tokensUsed?: TokenUsage;
}
```

## DI Container Contract

All components are registered via a DI container. The container must support:

```typescript
interface IContainer {
  register<T>(token: symbol, impl: new (...args: any[]) => T): void;
  resolve<T>(token: symbol): T;
}

// Tokens
const Tokens = {
  Scanner:          Symbol('Scanner'),
  PromptSanitizer:  Symbol('PromptSanitizer'),
  LLMClient:        Symbol('LLMClient'),
  CodeGenerator:    Symbol('CodeGenerator'),
  TokenTracker:     Symbol('TokenTracker'),
  Validator:        Symbol('Validator'),
  FileWriter:       Symbol('FileWriter'),
  Pipeline:         Symbol('Pipeline'),
} as const;
```

Container writes are **atomic** — partial updates must roll back.

---

## Security Properties

| Property | Mechanism |
|---|---|
| Prompt injection defense | `AbstractPromptSanitizer` — never autogen'd |
| Dangerous API blocking | `AbstractValidator.scanDangerousAPIs()` with configurable patterns |
| Contract enforcement | `AbstractValidator.verifyContract()` — all methods must be present |
| Compilation gate | `AbstractValidator.checkCompilation()` — TypeScript must compile |
| Budget circuit breaker | `AbstractTokenTracker.checkBudget()` — hard stop on cost overrun |
| Atomic writes | `AbstractFileWriter.write()` — tmp file + rename, no partial writes |
| Type safety chain | Branded types prevent skipping pipeline stages |
